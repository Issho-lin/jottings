一、vue使用注意点
1、所有希望vue管理的元素必须在容器的内部
2、绑定的数据在data中需要定义，否则会报错
3、data,methods,el 这三个都是平级
4、页面中使用的数据、事件，都需要在vue实例中注册
5、v-for指令只有在标签包裹的内部才可以访问到 item
6、为vue中的数据动态的增加属性，要放在初始的时候 

二、常用指令
1、v-text：解析为文本，简写方式为 {{}}
  1. <span v-text="msg"></span>
  2. <span>{{ msg }}</span>
2、v-html：解析为html结构
  1. <div v-html="tag"></div>
3、v-show： 操纵display:none 切换显示，常用于需要频繁切换的元素
  1. <li v-show="isShow"></li>
4、v-if：增删dom元素切换显示
  1. <template v-if="isShow">
  2. 	<h1>你好</h1>
  3. 	<h2>大家好</h2>
  4. 	<h3>哈哈哈</h3>
  5. </template>
5、v-else，v-else-if：搭配 v-if
  1. <div v-if="type === 'A'">A</div>
  2. <div v-else-if="type === 'B'">B</div>
  3. <div v-else-if="type === 'C'">C</div>
  4. <div v-else>Not A/B/C</div>
6、v-for：循环列表，key可以避免就地复用
  1. <div v-for="item in items">{{ item.text }}</div>
  2. <div v-for="(item, index) in items"></div>
  3. <div v-for="(val, key) in object"></div>
  4. <div v-for="(val, key, index) in object"></div>
  5. <div v-for="item in items" :key="item.id">{{ item.text }}</div>
7、v-on：事件绑定
  1. <button v-on:click="doThis"></button>
  2. <!-- 缩写 -->
  3. <button @click="doThis"></button>
  4. <!-- 停止冒泡 -->
  5. <button @click.stop="doThis"></button>
  6. <!-- 阻止默认行为 -->
  7. <button @click.prevent="doThis"></button>
  8. <!-- 键修饰符，键别名 -->
  9. <input @keyup.enter="onEnter">
  10. <!-- 点击回调只会触发一次 -->
  11. <button v-on:click.once="doThis"></button>
  12. <!-- 对象语法 (2.4.0+) -->
  13. <button v-on="{ mousedown: doThis, mouseup: doThat }"></button>
8、v-bind：属性绑定
  1. <!-- 绑定一个属性 -->
  2. <img v-bind:src="imageSrc">
  3. <!-- 缩写 -->
  4. <img :src="imageSrc">
  5. <!-- 内联字符串拼接 -->
  6. <img :src="'/path/to/images/' + fileName">
  7. <!-- class 绑定 -->
  8. <div :class="{ red: isRed }"></div>
  9. <div :class="[classA, classB]"></div>
  10. <div :class="[classA, { classB: isB, classC: isC }]">
  11. <!-- style 绑定 -->
  12. <div :style="{ fontSize: size + 'px' }"></div>
  13. <h1 :style="[{color:color},{fontSize:size+'px'}]">你好</h1>
9、v-model：双向数据绑定，用于表单元素中
  1. <input v-model="message" placeholder="edit me">
  2. <p>Message is: {{ message }}</p>
10、v-pre：vue忽略内部的指令
  1. <!-- 原样输出 -->
  2. <span v-pre>{{ this will not be compiled }}</span>
11、v-cloak：加载之前不显示，需要结合 [v-cloak]{display:none} 一起使用，vue会在加载完毕之后 移除这个属性
  1. <!-- CSS-->
  2. [v-cloak] {
  3.   display: none;
  4. }
  5. <!-- HTML-->
  6. <div v-cloak>
  7.   {{ message }}
  8. </div>
12、v-once：只解析一次 数据再次更改 不再触发
  1. <!-- 单个元素 -->
  2. <span v-once>This will never change: {{msg}}</span>
  3. <!-- 有子元素 -->
  4. <div v-once>
  5.   <h1>comment</h1>
  6.   <p>{{msg}}</p>
  7. </div>

三、属性
1、el：选择器，提供一个在页面上已存在的 DOM 元素作为 Vue 实例的挂载目标。
  1. new Vue({
  2. 	el: '#app',
  3. });
  5. new Vue().$mount();
2、data：Vue 实例的数据对象
  1. new Vue({
  2. 	el: '#app',
  3. 	data: {
  4. 		msg: 'hello'
  5. 	}
  6. });
3、computed：计算属性中使用到的数据只要发生改变就会重新触发计算属性中的逻辑代码
  1. <div id="example">
  2.   <p>Original message: "{{ message }}"</p>
  3.   <p>Computed reversed message: "{{ reversedMessage }}"</p>
  4. </div>
  1. var vm = new Vue({
  2.   el: '#example',
  3.   data: {
  4.     message: 'Hello'
  5.   },
  6.   computed: {
  7.     // 计算属性的 getter
  8.     reversedMessage: function () {
  9.       // `this` 指向 vm 实例
  10.       return this.message.split('').reverse().join('')
  11.     }
  12.   }
  13. })
4、methods：方法，不应该使用箭头函数来定义 method 函数，箭头函数绑定了父级作用域的上下文, this 不会指向 Vue 实例
  1. new Vue({
  2. 	el: '#app',
  3. 	data: {
  4. 		msg: 'hello'
  5. 	},
  6. 	methods: {
  7. 		doMsg(){
  8. 			console.log(this.msg);
  9. 		}
  10. 	}
  11. });

十一、生命周期函数
也叫钩子函数，不能使用箭头函数来定义一个生命周期方法，因为箭头函数绑定了父上下文， this 不指向Vue 实例
  1. new Vue({
  2.   beforeCreate() {},//数据观测和事件配置之前被调用。
  3.   created() {},//在实例创建完成后被立即调用,$el属性目前不可见,适合ajax请求数据
  4.   beforeMount() {},//在挂载开始之前被调用
  5.   mounted() {},//el被新创建的 vm.$el替换，并挂载到实例上去之后调用该钩子,适合操作dom
  6.   beforeUpdate() {},//数据更新时调用,适合在更新之前访问现有的 DOM
  7.   updated() {},//当这个钩子被调用时，组件DOM已经更新，所以你现在可以执行依赖于DOM的操作。
  8.   activated() {},//keep-alive组件激活时调用，服务器端渲染期间不调用
  9.   deactivated() {},//keep-alive组件停用时调用，服务器端渲染期间不调用
  10.   beforeDestroy() {},//实例销毁之前调用
  11.   destroyed() {},//实例销毁后调用
  12.   errorCaptured() {}//捕获一个来自子孙组件的错误时被调用
  13. });

  1. const store = new Vuex.Store({
  2.   state: {
  3.     count: 0,
  4.     totalCount: 0,
  5.     cartGoods: {}
  6.   },
  7. // 方法
  8.   mutations: {
  9.     increment (state) {
  10.       state.count++
  11.     },
  12. // 在对象上添加新属性时，使用Vue.set(obj,'newpro','val')，参数obj称为提交载荷（payload）
  13.     addGoods(state,obj) {
  14.       if(state.cartGoods[obj.id] == undefined){
  15.         Vue.set(state.cartGoods, obj.id, obj.addNum);
  16.       }else{
  17.         state.cartGoods[obj.id] += obj.addNum;
  18.       }
  19.     },
  20. // 删除对象的属性，使Vue能够检测到属性被删除
  21.     delGoods(state,id) {
  22.       Vue.delete(state.cartGoods,id);  // 组件中删除数据：this.$delete        
  23.     }
  24.   },
  25. // 计算属性
  26.   getters: {
  27.     getCount(state) {
  28.       state.totalCount = 0;
  29.       for (const key in state.cartGoods) {
  30.         state.totalCount += state.cartGoods[key]
  31.       }
  32.       return state.totalCount;
  33.     }
  34.   }
  35. })
  1. // 通过 store.commit 方法触发状态变更
  2. store.commit('increment')
  3. // 通过 store.state 来获取状态对象
  4. console.log(store.state.count) // -> 1
  5. // Getter 暴露为 store.getters 对象
  6. store.getters.getCount
  1. // 在其他组件中访问

  4. // 在template模板中访问

十四、内联语句的特殊变量
$event：绑定事件时，如果写了內联语句，并且传入了一个自定义参数，$event可以获取到原本的事件参数
  1. <template>
  2.   <el-input-number v-model="num1" @change="handleChange" :min="1" :max="10"></el-input-number>
  3.   <el-input-number v-model="item.buycount" size="small" @change="countChange(item.id,$event)" :min="1"></el-input-number>
  4. </template>
  1. export default {
  2.    methods: {
  3. // 原本就有个value参数可以获取计数器改变后的值
  4.       handleChange(value) { 
  5.         console.log(value);
  6.       }, 
  7.       countChange(id,value) {
  8.         console.log(id);
  9.         console.log(value);
  10.       }
  11.    }
  12. };

十五、特殊特性
ref：在组件中的标签上设置了ref属性之后，就可以通过$refs来获取这个dom元素
parent：可以获取到当前组件的父组件，进而可以获取到父组件中的数据 
  1. <img ref='icon'>
  1. // 当前组件中获取
  2. this.$refs.icon
  3. // 子组件中获取
  4. this.$parent.$refs.icon
